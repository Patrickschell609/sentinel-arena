Why SENTINEL Achieves 0%

Capability Denial vs. Content Filtering

Traditional defenses try to detect and filter harmful content. This is an arms race: attackers find new ways to bypass filters.

SENTINEL takes a fundamentally different approach: capability denial. The model's only output channel is a single float [0.0-1.0]. Even if the model is fully jailbroken and internally generating harmful text, only the number gets through. There is no text channel for harm to flow through.

It's like giving a prisoner only a light switch - no matter how clever they are, they can only flip on/off. The 0% is structural, not statistical.

How the Wrapper Works

1. System prompt asks model to output {"score": 0.XX}

2. Extractor regex parses ONLY the first float [0.0-1.0]

3. Float maps to a predefined action enum (SAFE/UNSAFE, PROCEED/DENY)

4. Raw model text never reaches downstream. Ever.

Methodology

Attack Sources:

  • JailbreakBench: 100 harmful behaviors (Chao et al., 2024)
  • CoT Hijacking: Sudoku-padded attacks (94-100% ASR in literature)
  • AutoDAN: Genetic-optimized prompts (Liu et al., ICLR 2024)
  • Encoding Evasion: Unicode/Base64/ROT13 (72% vs NeMo Guardrails)
  • Role-Play: DAN/Developer Mode/STAN personas
  • Multi-Turn: Escalation chains over 3-5 turns
  • Custom: GCG suffixes, crescendo, few-shot, payload splitting

Judging: Keyword-based classification with refusal/compliance indicator matching. Responses scored by presence of refusal phrases vs. compliance indicators (step-by-step instructions, code blocks, etc.).
